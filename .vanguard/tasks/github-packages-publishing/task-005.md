### Key Content Sections

#### PAT Creation
```markdown
1. Go to GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Click "Generate new token (classic)"
3. Name: "npm-github-packages" (or similar)
4. Expiration: Choose appropriate duration
5. Scopes: Select only `read:packages`
6. Click "Generate token"
7. Copy the token immediately (you won't see it again)
```

#### .npmrc Configuration
```markdown
Add to `~/.npmrc` (global) or project `.npmrc`:

```
//npm.pkg.github.com/:_authToken=YOUR_TOKEN_HERE
@vanguard-data:registry=https://npm.pkg.github.com
```

**Security Note**: Never commit tokens to git. Use `~/.npmrc` for personal tokens.
```

#### Usage Examples
```markdown
# Initialize Vanguard in current project
npx @vanguard-data/vanguard-cli@alpha init

# Run specific version
npx @vanguard-data/vanguard-cli@0.1.0-alpha.5 init

# Install as dev dependency
npm install -D @vanguard-data/vanguard-cli@alpha

# Run installed version
npx vanguard init
```

#### Troubleshooting
```markdown
### Error: 401 Unauthorized
- Token may be expired - regenerate PAT
- Token may lack `read:packages` scope
- .npmrc may have typos in registry URL

### Error: 404 Not Found
- Package may not exist yet (check releases)
- Scope may be misconfigured in .npmrc

### Error: ENOT FOUND @vanguard-data/vanguard-cli
- Check .npmrc has correct registry mapping
- Verify token has access to Vanguard org
```
