# Task 1: Create env var token utilities

> Infrastructure utilities for env var-based API token management.

## Context

**Feature**: Integration API Key Obfuscation
**Epic**: Security — remove plain text API keys from `.vanguard/integrations/*.json`

## Objective

Create utility functions for generating env var names, detecting env var references, resolving env vars, and reading/writing `.env` file entries. These are the foundation all other tasks depend on.

## Acceptance Criteria

- [ ] Function `generateEnvVarName(config)` produces `VANGUARD_{NAME}_TOKEN` (uppercased, hyphens to underscores)
- [ ] Function `isEnvVarReference(value)` detects strings with `env:` prefix
- [ ] Function `resolveEnvVarReference(value)` returns the resolved value from `process.env`, or throws if not set
- [ ] Function `readEnvFile(filePath)` parses a `.env` file into key-value pairs
- [ ] Function `writeEnvEntry(filePath, key, value)` appends or updates a single entry in a `.env` file
- [ ] No external dependencies added (no `dotenv` — use simple parser)

### Architecture Guidance

**Layer**: Infrastructure — these are file-system and environment concerns.

**Pattern**: Pure utility functions, no class needed.

**Env var naming convention**:
```
Input:  { name: "clickup-vanguard" }
Output: "VANGUARD_CLICKUP_COPOINT_AI_TOKEN"
```

**Env var reference format**:
```
"env:VANGUARD_CLICKUP_COPOINT_AI_TOKEN"
```

**`.env` file format** (standard):
```
# Vanguard integration tokens
VANGUARD_CLICKUP_COPOINT_AI_TOKEN=pk_111940232_xxx
```

**Key implementation details**:
- `generateEnvVarName`: `VANGUARD_${name.toUpperCase().replace(/-/g, '_')}_TOKEN`
- `isEnvVarReference`: `value.startsWith('env:')`
- `resolveEnvVarReference`: strip `env:` prefix, look up `process.env[varName]`
- `readEnvFile`: handle comments (`#`), empty lines, `KEY=VALUE` format, quoted values
- `writeEnvEntry`: read existing file, update if key exists, append if new, preserve comments
