### Architecture Guidance

**Layer**: Application (orchestration)

**Key flow change in `loadConfigurations()`**:
```typescript
private async loadConfigurations(): Promise<IntegrationConfig[]> {
  // Load .env file into process.env
  const envPath = path.join(process.cwd(), '.env')
  loadEnvFile(envPath)  // from dotenv utility (Task 1)

  const configs = /* ... existing file reading logic ... */

  // Resolve env var references
  return configs.map(config => {
    if (isEnvVarReference(config.apiToken)) {
      const resolvedToken = resolveEnvVarReference(config.apiToken)
      return { ...config, apiToken: resolvedToken }
    }
    return config
  })
}
```

**Backwards compatibility**: If `apiToken` does NOT start with `env:`, treat it as a raw token (existing behavior). This allows gradual migration.

**Error handling**: If `resolveEnvVarReference` throws because the env var isn't set, catch it and provide a helpful error message like:
```
Integration "clickup-vanguard" requires env var VANGUARD_CLICKUP_COPOINT_AI_TOKEN.
Set it in your .env file or environment.
```

**Note**: `loadEnvFile` should only set vars that aren't already in `process.env` (don't override explicit env vars).
