**Auto-migration logic** in `loadConfigurations()`:
```typescript
for (const config of configs) {
  if (!isEnvVarReference(config.apiToken)) {
    // Plain text token found — migrate it
    const envVarName = generateEnvVarName(config)
    await writeEnvEntry(envPath, envVarName, config.apiToken)

    const migratedConfig = { ...config, apiToken: `env:${envVarName}` }
    await this.saveConfiguration(migratedConfig)

    // Use resolved config for this session
    resolvedConfigs.push({ ...config })  // keep raw token in memory
  }
}
```

**Idempotency**: `isEnvVarReference()` check ensures already-migrated configs are skipped.

**Note**: The migration writes to both `.env` and the JSON file, so it needs write access. This happens transparently on any command that loads integrations (`list`, `test`, `initialize`, etc.).
