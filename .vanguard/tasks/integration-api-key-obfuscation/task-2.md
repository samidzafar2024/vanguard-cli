# Task 2: Update integration save flow to store env var reference

> Modify the save path so API tokens are stored as env var references, not plain text.

## Context

**Feature**: Integration API Key Obfuscation
**Epic**: Security — remove plain text API keys from `.vanguard/integrations/*.json`

## Objective

When an integration config is saved, replace the raw `apiToken` value with an `env:VAR_NAME` reference and write the actual token to the project's `.env` file.

## Acceptance Criteria

- [ ] `IntegrationService.saveConfiguration()` writes `apiToken: "env:VANGUARD_{NAME}_TOKEN..."` to JSON, not the raw token
- [ ] The actual token is written to the project root `.env` file
- [ ] If `.env` already has the key, it is updated (not duplicated)
- [ ] The `.env` file is created if it doesn't exist
- [ ] Before writing to `.env`, verify `.env` is listed in `.gitignore`; if not, warn the user or add it automatically
- [ ] Connection testing still works (uses raw token before saving)
- [ ] Tests pass: `npm run test`
- [ ] Lint passes: `npm run lint`

### Architecture Guidance

**Layer**: Application (orchestration) + Infrastructure (file I/O)

**Pattern**: The service orchestrates: generate env var name → write to `.env` → save config with reference.

**Key flow change in `IntegrationService.addIntegration()`**:
```typescript
// Current:
await this.saveConfiguration(config)

// New:
const envVarName = generateEnvVarName(config)
const envPath = path.join(process.cwd(), '.env')

// Safety check: ensure .env is gitignored before writing secrets
ensureEnvInGitignore(process.cwd())

await writeEnvEntry(envPath, envVarName, config.apiToken)
const obfuscatedConfig = { ...config, apiToken: `env:${envVarName}` }
await this.saveConfiguration(obfuscatedConfig)

// Still store provider with REAL token for current session
this.providers.set(config.name, provider)
```

**`.gitignore` check**: Read the project's `.gitignore` and verify `.env` is covered. If not, append `.env` to the file and log a message (e.g., `"Added .env to .gitignore to protect secrets"`). This prevents accidental token commits.

**Important**: The provider is already initialized with the real token before save. The obfuscation only affects what's persisted to disk.

**Note on `saveConfiguration`**: Currently in `IntegrationService`, it writes via `fileWriter`. The method should receive the already-obfuscated config. No changes needed to `FileIntegrationRepository.save()` — it just serializes what it receives.
