# Step 3/5: Define Data Models

> Design entities, value objects, and their relationships

## Objective

Design data models for the feature

## Inputs Required

- Component design

## Outputs Produced

- Entity definitions
- Value object definitions
- Relationship diagrams

---

Design data models:
- Identify entities (with identity and lifecycle)
- Identify value objects (immutable, compared by value)
- Define relationships and cardinality
- Specify validation rules

---

## Checkpoint

Before proceeding to the next step, confirm:

> **Are data models complete and correctly typed?**

## Tips

- Start with the domain model, then map to persistence
