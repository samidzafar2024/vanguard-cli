# Step 2/6: Define Personas

> Create user personas that guide requirement decisions

## Objective

Create clear user personas guiding requirement decisions

## Inputs Required

- Feature context from step 1

## Outputs Produced

- 2-4 user personas with goals and pain points

---

Create 2-4 user personas:
- Name and role
- Goals and motivations
- Pain points
- Technical proficiency
- Usage frequency

---

## Checkpoint

Before proceeding to the next step, confirm:

> **Are personas realistic? Do they cover all user types?**

## Tips

- Base personas on real user research when possible
