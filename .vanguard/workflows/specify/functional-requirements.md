# Step 4/6: Functional Requirements

> Create prioritized list of functional requirements

## Objective

Create prioritized list of functional requirements

## Inputs Required

- User scenarios

## Outputs Produced

- Prioritized functional requirements (must/should/could)

---

List all functional requirements using MoSCoW prioritization:
- MUST: Core functionality, non-negotiable
- SHOULD: Important but not critical
- COULD: Nice to have
- WON'T: Explicitly out of scope

---

## Checkpoint

Before proceeding to the next step, confirm:

> **Does each requirement trace back to a user scenario?**

## Tips

- Each requirement should be testable
- Trace each to a user scenario
