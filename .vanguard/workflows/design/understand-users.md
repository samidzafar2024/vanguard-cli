# Step 1/7: Understand Users

> Understand emotional user needs, not just functional

## Objective

Understand emotional needs beyond functional requirements

## Inputs Required

- Specification document
- User personas

## Outputs Produced

- Emotional needs map
- Experience goals

---

Facilitate discovery of emotional user needs:
- Ask what frustrates users about current solutions
- Understand what 'success' feels like for each persona
- Map emotional journey alongside functional journey

---

## Checkpoint

Before proceeding to the next step, confirm:

> **Do I understand how users should FEEL using this?**

## Tips

- Ask 'how should this feel?' not just 'what should it do?'
