# Technical Plan: GitHub Packages Publishing

> Automated npm package publishing to GitHub Packages for private distribution.

## Overview

This plan implements the GitHub Packages publishing feature as specified in `.vanguard/specs/github-packages-publishing.md`. The goal is to enable team members to install Vanguard CLI via `npx @vanguard-data/vanguard-cli@alpha init` with fully automated releases.

## Architecture Decision

**Approach**: Configuration-based publishing with GitHub Actions

This is purely an Infrastructure/DevOps change with no domain logic modifications required. The implementation focuses on:
1. Package.json configuration for GitHub Packages
2. Build configuration for clean npm publishing
3. Release channel management via file-based control
4. GitHub Actions workflow for automated CI/CD

## Component Breakdown

### 1. Package Configuration

**Purpose**: Configure package.json for `@vanguard-data` scoped publishing

**Changes**:
- Rename package from `vanguard-cli` to `@vanguard-data/vanguard-cli`
- Add `publishConfig` for GitHub Packages registry
- Add dual `bin` entries: `vanguard` and `vanguard-cli`
- Configure `repository` field for proper GitHub linking

**Files**:
- `package.json` (Modify)

### 2. Build Configuration

**Purpose**: Ensure clean TypeScript compilation for distribution

**Changes**:
- Verify `files` array includes all necessary assets
- Ensure shebang (`#!/usr/bin/env node`) in CLI entry point
- Validate `exports` configuration for ESM compatibility

**Files**:
- `package.json` (Modify)
- `src/presentation/cli/index.ts` (Verify shebang)
- `tsconfig.json` (Verify settings)

### 3. Release Channel Management

**Purpose**: Control release versioning via file-based channel selection

**Implementation**:
- Create `RELEASE_CHANNEL` file containing channel name (`alpha`, `beta`, `latest`)
- Version format: `{version}-{channel}.{increment}` (e.g., `0.1.0-alpha.1`)
- Workflow reads channel file to determine npm tag

**Files**:
- `RELEASE_CHANNEL` (Create) - Contains single word: `alpha`

### 4. GitHub Actions Workflow

**Purpose**: Fully automated release pipeline on push to main

**Workflow Steps**:
1. Checkout code
2. Setup Node.js 20.x
3. Install dependencies
4. Run quality gates (lint, typecheck, test)
5. Read release channel from file
6. Calculate next version number
7. Update package.json version
8. Build TypeScript
9. Publish to GitHub Packages
10. Create GitHub Release with notes
11. Commit version bump back to repo

**Files**:
- `.github/workflows/release.yml` (Create)

### 5. Documentation

**Purpose**: Provide clear setup instructions for internal developers

**Contents**:
- GitHub PAT creation guide
- `.npmrc` configuration
- Usage examples with version pinning
- Troubleshooting common auth issues

**Files**:
- `docs/INTERNAL_INSTALLATION.md` (Create)

## Risk Mitigation

| Risk | Mitigation |
|------|------------|
| Version collision | Workflow checks existing versions before publishing |
| Failed publish leaves inconsistent state | Workflow fails fast before version bump on errors |
| Token exposure | Use `GITHUB_TOKEN` (auto-provided, scoped to repo) |
